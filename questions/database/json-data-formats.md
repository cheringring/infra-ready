# JSON (JavaScript Object Notation) 기초

## JSON이란?

### 기본 개념
#### JSON의 정의:
- **JavaScript Object Notation**: JavaScript 객체 표기법에서 유래
- **경량 데이터 교환 형식**: XML보다 가볍고 읽기 쉬운 형태
- **텍스트 기반**: 사람이 읽고 쓰기 쉬운 텍스트 형식
- **언어 독립적**: JavaScript뿐만 아니라 모든 프로그래밍 언어에서 지원
- **표준화**: RFC 7159, ECMA-404 표준

#### JSON의 특징:
- **단순함**: 구문이 간단하고 이해하기 쉬움
- **가벼움**: XML 대비 적은 데이터 크기
- **파싱 속도**: 빠른 파싱과 생성 속도
- **웹 표준**: REST API의 사실상 표준 데이터 형식

## JSON 데이터 타입

### 기본 데이터 타입
#### 문자열 (String):
- **형식**: 큰따옴표로 감싸진 텍스트
- **예시**: `"안녕하세요"`, `"Hello World"`
- **이스케이프**: `\"`, `\\`, `\n`, `\t` 등 특수문자 처리

#### 숫자 (Number):
- **정수**: `42`, `-17`, `0`
- **실수**: `3.14`, `-0.5`, `1.23e-4`
- **특징**: 따옴표 없이 표기, JavaScript 숫자 형식 따름

#### 불린 (Boolean):
- **값**: `true` 또는 `false`
- **용도**: 참/거짓 상태 표현

#### null:
- **값**: `null`
- **의미**: 값이 없음을 명시적으로 표현
- **주의**: `undefined`는 JSON에서 지원하지 않음

### 복합 데이터 타입
#### 객체 (Object):
- **형식**: 중괄호 `{}` 안에 key-value 쌍
- **키**: 반드시 문자열 (큰따옴표 필수)
- **값**: 모든 JSON 데이터 타입 가능
- **구분**: 쉼표 `,`로 구분

#### 배열 (Array):
- **형식**: 대괄호 `[]` 안에 값들의 순서 있는 목록
- **요소**: 모든 JSON 데이터 타입 가능
- **구분**: 쉼표 `,`로 구분
- **인덱스**: 0부터 시작하는 순서

## JSON 구조 예시

### 간단한 객체
#### 사용자 정보:
- **이름**: `"name": "홍길동"`
- **나이**: `"age": 30`
- **활성 상태**: `"active": true`
- **주소**: `"address": null`

### 중첩된 객체
#### 회사 정보:
- **회사명**: `"company": "포인트아이"`
- **주소 객체**: `"address": { "city": "서울", "district": "강남구" }`
- **직원 배열**: `"employees": [{"name": "김철수"}, {"name": "이영희"}]`

### 복잡한 데이터 구조
#### API 응답 형태:
- **상태 코드**: `"status": 200`
- **메시지**: `"message": "성공"`
- **데이터 배열**: 여러 객체를 포함하는 배열
- **메타데이터**: 페이징, 총 개수 등 추가 정보

## JSON과 API 통신

### REST API에서의 JSON
#### HTTP 헤더:
- **Content-Type**: `application/json`
- **Accept**: `application/json`
- **요청/응답**: 모두 JSON 형식으로 데이터 교환

#### HTTP 메서드별 사용:
- **GET**: 응답 데이터를 JSON으로 반환
- **POST**: 요청 본문에 JSON 데이터 전송
- **PUT/PATCH**: 수정할 데이터를 JSON으로 전송
- **DELETE**: 삭제 확인 메시지를 JSON으로 응답

### JSON 파싱과 생성
#### JavaScript에서:
- **파싱**: `JSON.parse(jsonString)` - JSON 문자열을 객체로 변환
- **생성**: `JSON.stringify(object)` - 객체를 JSON 문자열로 변환
- **에러 처리**: `try-catch`로 파싱 오류 처리

#### Java에서:
- **Jackson 라이브러리**: `ObjectMapper`로 JSON 처리
- **Gson 라이브러리**: Google의 JSON 처리 라이브러리
- **어노테이션**: `@JsonProperty`, `@JsonIgnore` 등으로 매핑 제어

#### Python에서:
- **json 모듈**: `json.loads()`, `json.dumps()` 함수
- **딕셔너리 매핑**: Python dict와 JSON 객체 간 자동 변환
- **커스텀 인코더**: 복잡한 객체 직렬화 처리

## JSON 검증과 스키마

### JSON 문법 검증
#### 일반적인 오류:
- **따옴표 누락**: 키에 큰따옴표 필수
- **쉼표 오류**: 마지막 요소 뒤 쉼표 금지
- **중괄호/대괄호**: 열고 닫기 쌍 맞추기
- **데이터 타입**: undefined, function 등 지원하지 않는 타입

#### 검증 도구:
- **온라인 검증기**: JSONLint, JSON Formatter
- **IDE 플러그인**: 실시간 문법 검사
- **명령줄 도구**: `jq` 명령어로 JSON 검증 및 처리

### JSON Schema
#### 스키마 정의:
- **타입 검증**: 각 필드의 데이터 타입 정의
- **필수 필드**: `required` 배열로 필수 필드 지정
- **값 제한**: 최소/최대값, 문자열 길이, 패턴 등
- **중첩 구조**: 객체와 배열의 내부 구조 정의

#### 실무 활용:
- **API 문서화**: OpenAPI/Swagger에서 스키마 정의
- **데이터 검증**: 클라이언트/서버 간 데이터 무결성 보장
- **코드 생성**: 스키마로부터 자동 코드 생성

## JSON vs 다른 데이터 형식

### JSON vs XML
#### JSON 장점:
- **크기**: XML 대비 20-30% 작은 크기
- **파싱 속도**: 더 빠른 파싱 성능
- **가독성**: 사람이 읽기 쉬운 구조
- **JavaScript 호환**: 웹 개발에서 자연스러운 사용

#### XML 장점:
- **스키마 검증**: XSD를 통한 강력한 스키마 검증
- **네임스페이스**: 복잡한 문서 구조 지원
- **메타데이터**: 속성(attribute)을 통한 추가 정보
- **표준화**: 오래된 표준으로 다양한 도구 지원

### JSON vs YAML
#### YAML 장점:
- **가독성**: 들여쓰기 기반의 더 읽기 쉬운 형식
- **주석**: 주석 지원으로 문서화 가능
- **다중 문서**: 하나의 파일에 여러 문서 포함
- **설정 파일**: 설정 파일로 널리 사용

#### JSON 장점:
- **파싱 속도**: YAML보다 빠른 파싱
- **표준화**: 더 엄격한 표준과 광범위한 지원
- **크기**: 일반적으로 더 작은 파일 크기
- **API 표준**: REST API의 사실상 표준

## 실무에서의 JSON 활용

### 웹 개발에서의 JSON
#### AJAX 통신:
- **비동기 요청**: XMLHttpRequest, fetch API
- **데이터 교환**: 서버와 클라이언트 간 JSON 데이터 교환
- **DOM 업데이트**: JSON 데이터로 동적 페이지 업데이트

#### 설정 파일:
- **package.json**: Node.js 프로젝트 설정
- **tsconfig.json**: TypeScript 컴파일러 설정
- **launch.json**: VS Code 디버깅 설정

### 데이터베이스에서의 JSON
#### NoSQL 데이터베이스:
- **MongoDB**: BSON(Binary JSON) 형태로 문서 저장
- **CouchDB**: JSON 문서 기반 데이터베이스
- **Elasticsearch**: JSON 형태의 문서 검색

#### 관계형 데이터베이스:
- **MySQL**: JSON 데이터 타입 지원
- **PostgreSQL**: JSONB 타입으로 효율적인 JSON 저장
- **SQL Server**: JSON 함수들로 JSON 데이터 처리

### API 설계에서의 JSON
#### RESTful API:
- **요청/응답**: 표준 JSON 형식 사용
- **에러 응답**: 일관된 JSON 에러 형식
- **페이징**: JSON으로 페이징 정보 포함
- **HATEOAS**: 링크 정보를 JSON에 포함

#### GraphQL:
- **스키마**: JSON 형태의 스키마 정의
- **쿼리 결과**: JSON 형태로 결과 반환
- **변수**: JSON 형태로 쿼리 변수 전달

## JSON 보안 고려사항

### 보안 위험
#### JSON Injection:
- **위험**: 악의적인 JSON 데이터 주입
- **방어**: 입력 데이터 검증 및 이스케이프 처리
- **스키마 검증**: JSON Schema로 구조 검증

#### 대용량 JSON:
- **DoS 공격**: 매우 큰 JSON으로 서버 자원 고갈
- **방어**: 요청 크기 제한, 파싱 깊이 제한
- **스트리밍**: 대용량 JSON의 스트리밍 파싱

### 모범 사례
#### 데이터 검증:
- **타입 검증**: 예상되는 데이터 타입 확인
- **범위 검증**: 숫자 범위, 문자열 길이 검증
- **필수 필드**: 필수 필드 존재 여부 확인

#### 에러 처리:
- **파싱 오류**: JSON 파싱 실패 시 적절한 에러 응답
- **부분 실패**: 일부 데이터 처리 실패 시 대응 방안
- **로깅**: JSON 처리 과정의 적절한 로깅
