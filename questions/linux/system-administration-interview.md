# Linux 시스템 관리 - 포트폴리오 기반 답변

## 1. Linux 시스템 리소스 사용량 확인

### 포트폴리오 연결: 42 이노베이션 아카데미 + 데이터 사이언스 프로젝트

**"42 이노베이션 아카데미에서 Unix 시스템 프로그래밍과 시스템 콜을 학습하며 프로세스 관리의 기초를 익혔고, 데이터 사이언스 부트캠프에서 웹 스크래핑 프로젝트를 진행할 때 대용량 데이터 처리로 인한 시스템 부하를 실제로 모니터링하고 최적화했습니다."**

#### 실제 사용한 명령어들:
- **시스템 리소스 확인**: `top`, `htop`
- **CPU 분석**: `vmstat`, `iostat` (1초 간격 실시간 모니터링)
- **메모리 확인**: `free -h`, `/proc/meminfo` (사람이 읽기 쉬운 형태)
- **디스크 사용량**: `df -h`, `du -sh` (파일시스템별 사용량)
- **네트워크 모니터링**: `iftop`, `netstat -i` (실시간 트래픽 분석)
- **프로세스 특정**: `pgrep`, `top -p` (특정 프로세스 모니터링)

#### 실무 적용 사례:
- **스크래핑 프로세스 모니터링**: Python/Selenium 프로세스 상태 실시간 확인
- **리소스 사용량 추적**: 메모리, CPU, 디스크 사용량 정기 모니터링
- **자동화된 체크**: 스크립트를 통한 시스템 상태 자동 점검
- **임계값 알림**: 리소스 사용량이 임계치 초과 시 알림 시스템

## 2. 프로세스가 응답하지 않을 때 처리 방법

### 포트폴리오 연결: 42 이노베이션 아카데미 + 웹 스크래핑 프로젝트

**"42 이노베이션 아카데미에서 minishell 프로젝트를 통해 프로세스 생성, 관리, 시그널 처리를 직접 구현했고, 웹 스크래핑 프로젝트에서 Selenium WebDriver가 종종 응답하지 않는 문제를 체계적으로 해결한 경험이 있습니다."**

#### 42 이노베이션 아카데미에서 학습한 프로세스 관리 개념:
- **minishell 프로젝트**: 시그널 처리(`SIGINT`, `SIGQUIT`) 및 프로세스 생성/관리
- **핵심 시스템 콜**: `fork()`, `execve()`, `waitpid()`, `kill()`, `signal()`
- **프로세스 상태**: 실행, 대기, 좀비 프로세스 처리 방법 학습

#### 단계별 문제 해결 과정:
- **1단계 - 프로세스 식별**: `ps aux`, `pgrep` 명령어로 문제 프로세스 찾기
- **2단계 - 상태 확인**: `ps -eo` 옵션으로 프로세스 상태(R, S, D, Z, T) 분석
- **3단계 - 리소스 분석**: `top -p`, `lsof -p`로 특정 프로세스 리소스 사용량 확인
- **4단계 - 단계적 종료**: `kill` (SIGTERM) → `kill -9` (SIGKILL) 순서로 안전한 종료
- **5단계 - 관련 프로세스 정리**: `pkill` 명령어로 관련 프로세스 일괄 정리

#### 실제 구현한 자동 복구 시스템:
- **임계값 모니터링**: 메모리 2GB, CPU 80% 초과 시 자동 대응
- **프로세스 재시작**: 정상 종료 시도 후 강제 종료로 안전한 복구
- **로깅 시스템**: 모든 프로세스 관리 작업에 대한 상세 로그 기록
- **예외 처리**: 프로세스 접근 권한 문제 등 예외 상황 대응

## 3. crontab을 이용한 작업 스케줄링

### 포트폴리오 연결: 자동화된 데이터 수집 파이프라인

**"데이터 사이언스 프로젝트에서 정기적인 웹 데이터 수집을 위해 crontab을 활용한 자동화 시스템을 구축했습니다."**

#### 실제 구현한 crontab 설정:
- **일일 작업**: 매일 오전 9시 상품 정보 수집 (`0 9 * * *`)
- **주간 작업**: 매주 월요일 새벽 2시 카테고리 업데이트 (`0 2 * * 1`)
- **시간별 작업**: 매시간 정각 시스템 상태 체크 (`0 * * * *`)
- **실시간 모니터링**: 5분마다 프로세스 모니터링 (`*/5 * * * *`)
- **로그 관리**: 매일 자정 7일 이상 된 로그 파일 삭제 (`0 0 * * *`)
- **백업 작업**: 매월 1일 새벽 3시 데이터베이스 백업 (`0 3 1 * *`)

#### 실제 스크래핑 스크립트 구성:
- **환경 설정**: PATH, PYTHONPATH 환경변수 설정
- **로그 관리**: 로그 디렉토리 생성 및 시작/종료 시간 기록
- **가상환경**: Python 가상환경 활성화로 의존성 격리
- **에러 처리**: 성공/실패 상태에 따른 다른 알림 발송
- **결과 저장**: JSON 형태로 데이터 저장 후 데이터베이스 연동

#### crontab 관리 및 모니터링:
- **작업 확인**: `crontab -l` 명령어로 현재 설정된 스케줄 확인
- **서비스 상태**: `systemctl status cron`으로 cron 데몬 상태 점검
- **로그 분석**: `/var/log/cron`, `/var/log/syslog`에서 실행 이력 확인
- **문법 검증**: crontab.guru 같은 온라인 도구로 스케줄 표현식 검증
- **디버깅**: 특정 작업의 실행 결과 및 에러 로그 추적
