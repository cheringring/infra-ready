question: "GPU Sharing 프로젝트에서 NFS 기반 공유 스토리지를 구축한 이유와 장단점은?"
shortAnswer: "NFS를 선택한 이유는 여러 GPU 서버가 동시에 학습 데이터에 접근해야 하고, 중앙 집중식 데이터 관리가 필요했기 때문입니다. 장점은 설정이 간단하고 파일 시스템 수준에서 투명하게 작동한다는 것이며, 단점은 네트워크 지연과 단일 장애점(SPOF) 위험입니다. 이를 보완하기 위해 사용자별 데이터 격리와 권한 관리를 적용했습니다."

## 상세 답변

### NFS 선택 이유

#### 요구사항
- 여러 GPU 서버의 동시 데이터 접근
- 학습 데이터 중앙 관리
- 결과물 공유 및 백업

#### NFS의 적합성
- 파일 시스템 수준 공유
- 클라이언트 투명성
- 리눅스 네이티브 지원

### 구현 내용

#### 중앙 저장소 (NAS) 구축
- NFS 서버 설정
- 대용량 스토리지 구성
- 백업 정책 수립

#### 사용자별 데이터 격리
- 각 사용자 전용 디렉토리
- UID/GID 기반 권한 관리
- 쿼터 설정으로 용량 제한

#### 네트워크 연결 제어
- 리버스 프록시 구성
- 게이트웨이를 통한 접근
- 외부 접속 보안 강화

### 장단점 분석

#### 장점
- **설정 간단**: 표준 프로토콜
- **투명성**: 로컬 파일처럼 사용
- **호환성**: 모든 리눅스 지원
- **중앙 관리**: 백업/복구 용이

#### 단점
- **네트워크 지연**: 로컬보다 느림
- **SPOF**: NFS 서버 장애 시 전체 영향
- **동시성 제한**: 대량 쓰기 시 병목
- **보안**: 네트워크 스니핑 위험

### 개선 방안

#### 성능 최적화
- SSD 기반 스토리지
- 10Gbps 네트워크
- 캐싱 전략

#### 고가용성
- NFS 서버 이중화
- 자동 Failover
- 정기 백업

#### 보안 강화
- Kerberos 인증
- NFSv4 암호화
- 방화벽 규칙

### 대안 기술

**Ceph/GlusterFS**: 분산 파일 시스템
**S3 호환 스토리지**: 객체 스토리지
**iSCSI**: 블록 스토리지

### 실무 적용

이 경험을 통해 공유 스토리지 아키텍처, 네트워크 파일 시스템, 멀티 유저 환경 관리 역량을 습득했습니다.
