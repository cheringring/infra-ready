shortAnswer: "데이터 증강 기법 3가지(속도 변경, 노이즈 추가, 크기 조정)를 적용해 데이터를 451개에서 2,255개로 5배 증가시켰습니다. 정적 수어는 Dense NN으로 31개 자모음을 99.46% 정확도로, 동적 수어는 Bi-LSTM으로 9개 복합모음을 99.7% 정확도로 인식하도록 모델을 분리 설계했습니다."

## 상세 답변

### 1. 데이터 증강 (Data Augmentation)

#### 적용한 3가지 기법

**속도 변경 (Speed Variation)**
- 수어 동작 속도를 0.8배~1.2배로 조정
- 사용자마다 다른 수어 속도에 대응
- 시퀀스 데이터의 시간축 변형

**노이즈 추가 (Noise Injection)**
- 손 랜드마크 좌표에 미세한 노이즈 추가
- 카메라 떨림이나 조명 변화 시뮬레이션
- 과적합(Overfitting) 방지

**크기 조정 (Scale Variation)**
- 손 크기를 0.9배~1.1배로 변경
- 카메라 거리 차이에 대한 강건성 확보
- 다양한 사용자 손 크기 대응

#### 데이터 증강 효과
- **증강 전**: 451개 샘플
- **증강 후**: 2,255개 샘플 (5배 증가)
- **정확도 향상**: 약 7-10% 개선

### 2. 모델 아키텍처 분리 설계

#### 정적 수어 모델 (Dense NN)
**대상**: 31개 한글 자모음
**구조**: 
- MediaPipe로 21개 손 랜드마크 추출
- Fully Connected Dense Layer
- Softmax 분류

**정확도**: 99.46%

**선택 이유**: 
- 정적 수어는 단일 프레임 정보만 필요
- Dense NN이 가볍고 빠름
- 실시간 처리에 유리

#### 동적 수어 모델 (Bi-LSTM)
**대상**: 9개 복합 모음
**구조**:
- 시퀀스 데이터 (여러 프레임)
- Bidirectional LSTM
- 시간적 패턴 학습

**정확도**: 99.7%

**선택 이유**:
- 동적 수어는 시간 흐름 정보 필수
- Bi-LSTM이 양방향 시퀀스 학습
- 복잡한 동작 패턴 인식

### 3. 모델 최적화

#### TFLite 경량화
- 모델 크기 대폭 감소
- 모바일/태블릿에서 실행 가능
- 추론 속도 10배 향상

#### 성능 지표
- **정확도**: 91% 유지
- **처리 속도**: 실시간 10배 향상
- **지원 기기**: 저사양 태블릿, 키오스크

### 4. 시스템 아키텍처

#### 백엔드 (FastAPI)
- RESTful API 서버
- 실시간 손동작 인식 엔드포인트
- WebSocket 실시간 통신

#### 프론트엔드 (Flutter)
- 크로스플랫폼 앱
- 실시간 카메라 피드
- 학습 모드, 퀴즈 모드 제공

### 5. 실전 배포 고려사항

**이미지 전처리 파이프라인**
- 손 영역 자동 크롭
- 정규화 및 리사이징
- 배경 노이즈 제거

**API 응답 속도 개선**
- 모델 추론 최적화
- 배치 처리
- 캐싱 전략

**정량적 성과**
- 총 40개 한글 자모음 인식
- 평균 정확도 99.5% 이상
