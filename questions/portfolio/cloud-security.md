question: "네이버 클라우드 보안 전문가 과정에서 학습한 IAM과 ACG를 실무에 어떻게 적용하시겠습니까?"
shortAnswer: "IAM으로 최소 권한 원칙을 적용해 각 사용자/서비스에 필요한 권한만 부여하고, ACG(Access Control Group)와 Network ACL을 조합한 이중 보안 체계로 네트워크 계층과 인스턴스 계층을 모두 보호하겠습니다. 예를 들어 웹 서버는 80/443 포트만, DB 서버는 특정 IP에서만 3306 포트 접근을 허용하는 식으로 구성합니다."

## 상세 답변

### IAM (Identity and Access Management)

#### 최소 권한 원칙 적용
- 각 사용자/서비스에 필요한 최소한의 권한만 부여
- 역할 기반 접근 제어 (RBAC)
- 정기적인 권한 검토 및 회수

#### 실무 적용 예시
**개발팀**: 개발 환경 리소스만 접근
**운영팀**: 프로덕션 환경 읽기 전용
**관리자**: 전체 권한 + MFA 필수

### ACG와 Network ACL 이중 보안

#### ACG (Access Control Group)
- 인스턴스 수준 방화벽
- Stateful (상태 저장)
- 허용 규칙만 설정 가능

#### Network ACL
- 서브넷 수준 방화벽
- Stateless (상태 비저장)
- 허용/거부 규칙 모두 가능

#### 이중 보안 체계 구성
**1단계**: Network ACL로 서브넷 전체 보호
**2단계**: ACG로 개별 인스턴스 세밀 제어

### ISMS-P 준수

학습한 관리적·기술적 보호조치를 실무에 적용하여 법적 규제를 준수하고 개인정보를 안전하게 관리하겠습니다.
