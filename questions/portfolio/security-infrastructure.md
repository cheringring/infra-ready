shortAnswer: "SSH 포트 변경과 root 로그인 금지가 가장 중요합니다. 기본 22번 포트는 무작위 대입 공격의 주요 타겟이며, root 직접 접속을 차단하면 공격자가 관리자 계정명을 추측해야 하므로 보안이 2중으로 강화됩니다. 실제로 4242 포트로 변경하고 sudo 그룹만 권한을 부여해 공격 표면을 최소화했습니다."

## 상세 답변

### 가장 중요한 보안 정책: SSH 강화

#### 1. SSH 포트 변경 (22 → 4242)

**중요한 이유**
- 기본 22번 포트는 자동화된 봇 공격의 1순위 타겟
- 비표준 포트 사용으로 무작위 스캔 공격 회피
- 로그에서 실제 공격 시도와 일반 스캔을 구분 가능

**구현 방법**
```bash
# /etc/ssh/sshd_config
Port 4242
```

#### 2. Root 로그인 금지

**중요한 이유**
- root는 모든 시스템에 존재하는 계정명 (추측 불필요)
- root 직접 접속 차단 시 공격자는 일반 계정명도 알아야 함
- 2단계 인증 효과 (일반 계정 로그인 → sudo 권한 상승)

**구현 방법**
```bash
# /etc/ssh/sshd_config
PermitRootLogin no
```

### 기타 중요 보안 정책

#### Password Policy
**libpam-pwquality 설정**
- 최소 10자, 대문자/숫자/특수문자 포함
- 이전 비밀번호 재사용 금지
- 사용자명 포함 금지

**효과**: 사전 공격(Dictionary Attack) 방어

#### Sudo 권한 통제
**설정 내용**
- sudo 그룹만 관리자 권한 부여
- 모든 sudo 명령 로그 기록 (TTY)
- 실패한 sudo 시도도 기록

**효과**: 권한 남용 추적 및 감사 가능

#### UFW 방화벽
**설정 내용**
- 4242 포트 외 모든 인바운드 차단
- 아웃바운드는 필요한 것만 허용

**효과**: 불필요한 서비스 노출 방지

### LVM 스토리지 관리

**보안 관점의 장점**
- 암호화된 파티션 구성 가능
- 데이터 격리 및 용량 제한
- 서비스 중단 없는 확장

### 자동화 모니터링

**Bash 스크립트 + Cron**
- 10분마다 시스템 상태 브로드캐스팅
- 비정상 접속 즉시 감지
- CPU/메모리/디스크 이상 징후 파악

### 실무 적용 가능성

이러한 보안 설정은 실제 엔터프라이즈 환경의 베이스라인 보안 정책과 일치하며, ISMS-P 인증 기준의 기술적 보호조치 요구사항을 충족합니다.

### 추가 개선 방안

- Fail2ban: 반복 로그인 실패 시 IP 자동 차단
- 2FA: Google Authenticator 등 이중 인증
- SELinux/AppArmor: 강제 접근 제어
