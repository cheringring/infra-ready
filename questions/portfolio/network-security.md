## 상세 답변

### VLAN (Virtual LAN) 구현

#### 구현 내용

**논리적 네트워크 분리**
- 물리적으로 같은 스위치에 연결되어 있어도 논리적으로 분리
- 각 VLAN은 독립적인 브로드캐스트 도메인
- VLAN 10: 관리부서, VLAN 20: 개발부서, VLAN 30: 게스트

**Trunking 설정**
- 802.1Q 프로토콜 사용
- 여러 VLAN 트래픽을 하나의 물리적 링크로 전송
- VLAN 태그를 프레임에 추가하여 구분

#### 기술적 효과

**보안 강화**
- 부서 간 트래픽 격리
- 민감한 데이터 접근 제어
- 네트워크 세그멘테이션

**성능 개선**
- 브로드캐스트 트래픽 감소
- 네트워크 혼잡도 완화
- 대역폭 효율적 사용

**관리 용이성**
- 물리적 재배치 없이 논리적 이동
- 유연한 네트워크 구성
- 중앙 집중식 관리

### OSPF (Open Shortest Path First) 구현

#### 구현 내용

**동적 라우팅 프로토콜**
- Link-State 알고리즘 기반
- Dijkstra 알고리즘으로 최단 경로 계산
- Area 개념으로 대규모 네트워크 관리

**고가용성 확보**
- 링크 장애 시 자동 경로 재계산
- 수 초 내 우회 경로 설정
- 데이터 손실 최소화

**메트릭 기반 경로 선택**
- 대역폭을 고려한 Cost 계산
- 최적 경로 자동 선택
- 로드 밸런싱 가능

#### Static vs Dynamic Routing 비교

**Static Routing**
- 장점: 설정 간단, 리소스 적게 사용
- 단점: 장애 시 수동 대응, 확장성 낮음
- 용도: 소규모 네트워크, 고정 경로

**Dynamic Routing (OSPF)**
- 장점: 자동 장애 복구, 확장성 높음
- 단점: 설정 복잡, CPU/메모리 사용
- 용도: 대규모 네트워크, 가용성 중요

### 서브네팅 (Subnetting)

#### 구현 내용

**IP 주소 효율적 배분**
- CIDR 표기법 사용
- 네트워크 크기에 맞게 서브넷 분할
- IP 낭비 최소화

**예시**
```
본사: 192.168.1.0/24 (254 hosts)
지사1: 192.168.2.0/25 (126 hosts)
지사2: 192.168.2.128/25 (126 hosts)
```

### 실무 활용 사례

#### 1. 기업 네트워크 설계

**부서별 VLAN 분리**
- 경영진: VLAN 10 (높은 보안)
- 개발팀: VLAN 20 (인터넷 제한)
- 영업팀: VLAN 30 (외부 접속 허용)
- 게스트: VLAN 99 (격리)

**OSPF 이중화**
- 본사-지사 간 다중 경로
- 주 회선 장애 시 백업 회선 자동 전환
- 비즈니스 연속성 보장

#### 2. 데이터센터 네트워크

**Spine-Leaf 아키텍처**
- VLAN으로 테넌트 격리
- OSPF로 동적 경로 관리
- 고가용성 및 확장성

#### 3. 클라우드 VPC 설계

**AWS VPC 예시**
- 퍼블릭 서브넷: 웹 서버 (VLAN 개념)
- 프라이빗 서브넷: DB 서버
- 라우팅 테이블: OSPF와 유사한 동적 라우팅

#### 4. 보안 강화

**네트워크 세그멘테이션**
- DMZ: 외부 접근 서버
- 내부망: 업무 시스템
- 관리망: 인프라 관리

**ACL과 방화벽**
- VLAN 간 트래픽 제어
- 최소 권한 원칙 적용

### 학습 성과

**네트워크 설계 역량**
- 논리적 네트워크 구조 설계
- 트래픽 흐름 최적화
- 장애 대응 시나리오 수립

**실무 준비도**
- 엔터프라이즈 네트워크 이해
- 클라우드 네트워킹 기초
- 보안 네트워크 아키텍처

### 추가 학습 방향

- BGP: 인터넷 라우팅 프로토콜
- SD-WAN: 소프트웨어 정의 WAN
- Network Automation: Ansible, Terraform
