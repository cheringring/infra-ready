# 포인트아이 Java 개발자 면접 대비

## 포인트아이 회사 특성에 맞는 Java 기술 스택

### 통신사 인프라 개발 요구사항
#### 대용량 트래픽 처리:
- **동시 접속자**: 수만~수십만 명 동시 처리 필요
- **응답 시간**: 평균 100ms 이하 응답 속도 요구
- **가용성**: 99.9% 이상 서비스 가용성 보장
- **확장성**: 트래픽 증가에 따른 수평적 확장 가능

#### 공공기관 솔루션 특성:
- **보안**: 개인정보보호법, 정보보안 관리체계 준수
- **안정성**: 24시간 무중단 서비스 운영
- **호환성**: 레거시 시스템과의 연동 필요
- **표준화**: 정부 표준 프레임워크 준수

## Spring Boot 기반 엔터프라이즈 개발

### 대용량 트래픽 처리를 위한 최적화
#### 커넥션 풀 튜닝:
- **HikariCP**: `maximum-pool-size=20`, `minimum-idle=10`
- **커넥션 타임아웃**: `connection-timeout=30000ms`
- **유휴 타임아웃**: `idle-timeout=600000ms`
- **최대 수명**: `max-lifetime=1800000ms`

#### 캐싱 전략:
- **Redis**: 세션 저장, API 응답 캐싱
- **Spring Cache**: `@Cacheable`, `@CacheEvict` 어노테이션
- **로컬 캐시**: Caffeine, EhCache 활용
- **캐시 무효화**: TTL 설정, 수동 무효화 전략

#### 비동기 처리:
- **`@Async`**: 비동기 메서드 실행
- **CompletableFuture**: 비동기 작업 체이닝
- **Spring WebFlux**: 리액티브 프로그래밍
- **메시지 큐**: RabbitMQ, Kafka 활용

### 보안 강화
#### Spring Security 설정:
- **JWT 토큰**: 무상태 인증 방식
- **CORS 설정**: 크로스 오리진 요청 제어
- **CSRF 보호**: 사이트 간 요청 위조 방지
- **SQL Injection 방지**: PreparedStatement, JPA 활용

#### 데이터 암호화:
- **AES 암호화**: 민감한 데이터 암호화 저장
- **BCrypt**: 비밀번호 해싱
- **HTTPS**: SSL/TLS 통신 암호화
- **API 키 관리**: 환경 변수, Vault 활용

## JVM 성능 튜닝 실무

### 메모리 최적화
#### 힙 메모리 설정:
- **초기 힙**: `-Xms4g` (물리 메모리의 25%)
- **최대 힙**: `-Xmx8g` (물리 메모리의 50%)
- **Young Generation**: `-XX:NewRatio=2` (전체 힙의 1/3)
- **Survivor 영역**: `-XX:SurvivorRatio=8`

#### GC 튜닝:
- **G1GC 사용**: `-XX:+UseG1GC` (대용량 힙에 적합)
- **GC 목표 시간**: `-XX:MaxGCPauseMillis=200`
- **병렬 GC 스레드**: `-XX:ParallelGCThreads=8`
- **동시 GC 스레드**: `-XX:ConcGCThreads=2`

### 모니터링과 진단
#### JVM 모니터링 도구:
- **`jstat -gc [pid] 1s`**: 실시간 GC 통계
- **`jmap -dump:format=b,file=heap.hprof [pid]`**: 힙 덤프 생성
- **`jstack [pid]`**: 스레드 덤프 생성
- **VisualVM**: GUI 기반 JVM 모니터링

#### APM 도구 활용:
- **Spring Boot Actuator**: 애플리케이션 메트릭
- **Micrometer**: Prometheus, Grafana 연동
- **Zipkin**: 분산 트레이싱
- **New Relic, AppDynamics**: 상용 APM 솔루션

## Docker 기반 배포 전략

### 컨테이너 최적화
#### 이미지 경량화:
- **Alpine Linux**: 5MB 크기의 경량 베이스 이미지
- **멀티스테이지 빌드**: 빌드 도구 제외로 크기 50% 절약
- **레이어 캐싱**: 자주 변경되지 않는 의존성을 하위 레이어에 배치
- **불필요한 파일 제거**: `.dockerignore` 활용

#### 보안 강화:
- **비root 사용자**: `USER 1001` 또는 전용 사용자 생성
- **읽기 전용 파일시스템**: `--read-only` 옵션
- **보안 스캔**: Trivy, Clair로 취약점 검사
- **최소 권한**: 필요한 포트만 노출

### 운영 환경 배포
#### Blue-Green 배포:
- **무중단 배포**: 새 버전을 별도 환경에 배포 후 트래픽 전환
- **롤백 전략**: 문제 발생 시 즉시 이전 버전으로 복구
- **헬스 체크**: `/actuator/health` 엔드포인트 활용
- **로드 밸런서**: Nginx, HAProxy로 트래픽 분산

#### 컨테이너 오케스트레이션:
- **Docker Swarm**: 간단한 클러스터 관리
- **Kubernetes**: 대규모 컨테이너 오케스트레이션
- **서비스 메시**: Istio, Linkerd로 마이크로서비스 통신 관리
- **자동 스케일링**: CPU, 메모리 사용률 기반 확장

## 마이크로서비스 아키텍처

### 서비스 분해 전략
#### 도메인 주도 설계(DDD):
- **바운디드 컨텍스트**: 업무 도메인별 서비스 분리
- **애그리게이트**: 데이터 일관성 경계 정의
- **이벤트 스토밍**: 비즈니스 프로세스 분석
- **서비스 크기**: 2-Pizza 팀 규칙 (8명 이하)

#### 데이터 관리:
- **Database per Service**: 서비스별 독립적인 데이터베이스
- **CQRS**: 명령과 조회 책임 분리
- **Event Sourcing**: 이벤트 기반 데이터 저장
- **Saga 패턴**: 분산 트랜잭션 관리

### 서비스 간 통신
#### 동기 통신:
- **REST API**: HTTP 기반 RESTful 서비스
- **GraphQL**: 클라이언트 맞춤형 데이터 조회
- **gRPC**: 고성능 RPC 프레임워크
- **Circuit Breaker**: 장애 전파 방지

#### 비동기 통신:
- **Message Queue**: RabbitMQ, Apache Kafka
- **Event Bus**: 도메인 이벤트 발행/구독
- **Webhook**: HTTP 콜백 기반 알림
- **Server-Sent Events**: 실시간 데이터 푸시
