# OSI 7계층 모델

## 계층별 상세 설명

### 7계층 - 응용 계층 (Application Layer)
- **역할**: 사용자와 직접 상호작용하는 애플리케이션 서비스
- **프로토콜**: HTTP, HTTPS, FTP, SMTP, DNS, DHCP
- **예시**: 웹 브라우저, 이메일 클라이언트

### 6계층 - 표현 계층 (Presentation Layer)
- **역할**: 데이터 암호화, 압축, 인코딩/디코딩
- **기능**: SSL/TLS, 데이터 형식 변환
- **예시**: JPEG, GIF, ASCII, EBCDIC

### 5계층 - 세션 계층 (Session Layer)
- **역할**: 세션 설정, 관리, 종료
- **기능**: 대화 제어, 동기화
- **프로토콜**: NetBIOS, RPC, SQL

### 4계층 - 전송 계층 (Transport Layer)
- **역할**: 종단 간 신뢰성 있는 데이터 전송
- **프로토콜**: TCP, UDP
- **기능**: 오류 검출, 흐름 제어, 포트 번호 관리

### 3계층 - 네트워크 계층 (Network Layer)
- **역할**: 라우팅, 경로 선택
- **프로토콜**: IP, ICMP, ARP, OSPF, BGP
- **장비**: 라우터, L3 스위치

### 2계층 - 데이터링크 계층 (Data Link Layer)
- **역할**: 프레임 단위 데이터 전송, 오류 검출
- **프로토콜**: Ethernet, PPP, Wi-Fi
- **장비**: 스위치, 브리지
- **주소**: MAC 주소

### 1계층 - 물리 계층 (Physical Layer)
- **역할**: 물리적 신호 전송
- **매체**: 케이블, 광섬유, 무선
- **장비**: 허브, 리피터, 케이블

## 신입 면접 핵심 포인트

### 1. 캡슐화 과정
- 상위 계층에서 하위 계층으로 데이터 전달 시 헤더 추가
- 응용 → 표현 → 세션 → 전송 → 네트워크 → 데이터링크 → 물리

### 2. 실무 연관성
- **웹 서비스**: HTTP(7계층) → TCP(4계층) → IP(3계층) → Ethernet(2계층)
- **문제 해결**: 계층별로 문제 원인 분석 가능
