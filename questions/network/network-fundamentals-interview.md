# 네트워크 기초 - 포트폴리오 기반 답변

## 1. TCP와 UDP의 차이점

### 포트폴리오 연결: 42 이노베이션 아카데미 + KISA 보안 교육

**"42 이노베이션 아카데미에서 C 언어로 네트워크 프로그래밍을 구현하며 소켓 통신의 기초를 익혔고, KISA 정보보호 인력 양성 교육에서 Wireshark를 활용해 실제 네트워크 트래픽을 분석하며 TCP와 UDP의 동작 원리를 직접 확인했습니다."**

#### 실습에서 확인한 주요 차이점:

| 구분 | TCP | UDP |
|------|-----|-----|
| **연결성** | 연결 지향 (3-way handshake) | 비연결 지향 |
| **신뢰성** | 신뢰성 보장 (재전송, 순서 보장) | 신뢰성 보장 안함 |
| **속도** | 상대적으로 느림 | 빠름 |
| **헤더 크기** | 20바이트 | 8바이트 |
| **사용 예시** | HTTP, HTTPS, SSH, FTP | DNS, DHCP, 스트리밍 |

#### 웹 스크래핑 프로젝트에서의 적용:
- **TCP 사용**: 중요한 데이터 수집 시 신뢰성 보장 (HTTP 요청)
- **UDP 사용**: 모니터링 데이터 전송 시 속도 우선 (로깅)

## 2. 통신사 인프라에서 중요한 네트워크 프로토콜

### 포트폴리오 연결: NCP 보안 전문가 과정 - 클라우드 네트워크 아키텍처 학습

**"네이버 클라우드 보안 전문가 과정에서 대규모 클라우드 인프라의 네트워크 프로토콜을 학습했으며, 이는 통신사 인프라와 유사한 구조입니다."**

#### 통신사 핵심 프로토콜들:
- **SIP (Session Initiation Protocol)**: VoIP 통화 세션 설정 (포트 5060/5061)
- **RTP (Real-time Transport Protocol)**: 실시간 미디어 전송 (동적 포트)
- **DIAMETER**: 사용자 인증, 과금, 정책 관리 (포트 3868)
- **GTP (GPRS Tunneling Protocol)**: 모바일 데이터 터널링 (포트 2123/2152)
- **SNMP**: 네트워크 장비 모니터링 및 관리 (포트 161)

#### NCP 과정에서 학습한 관련 기술:
- **VPC 아키텍처**: Load Balancer, Network ACL, Auto Scaling 구성
- **프로토콜 지원**: HTTP/HTTPS, TCP/UDP 트래픽 처리
- **헬스 체크**: TCP/HTTP 기반 상태 모니터링
- **보안 규칙**: 인바운드/아웃바운드 트래픽 제어

#### 통신사 인프라 특성 이해:
- **고가용성**: 24시간 무중단 서비스 요구사항
- **대용량 처리**: 동시 다수 사용자 지원
- **실시간 모니터링**: 통화 품질, 패킷 손실률, 지연시간 측정
- **프로토콜 다양성**: SIP, RTP, DIAMETER 등 전용 프로토콜 사용

## 3. 방화벽 설정 경험

### 포트폴리오 연결: NCP 보안 전문가 과정 - ACG & Network ACL 실습

**"네이버 클라우드 보안 전문가 과정에서 ACG(Access Control Group)와 Network ACL을 이용한 다층 보안 체계를 직접 구축했습니다."**

#### 실제 구현한 보안 설정:
- **NCP ACG 설정**: 인스턴스 레벨에서 HTTP(80), HTTPS(443) 포트 허용
- **SSH 접근 제한**: 관리자 네트워크(특정 IP 대역)에서만 SSH(22) 접근 허용
- **프로토콜별 규칙**: TCP/UDP 프로토콜별로 세분화된 접근 제어

#### Network ACL 설정 (서브넷 레벨):
- **우선순위 기반 규칙**: 낮은 번호가 높은 우선순위
- **허용 규칙**: 웹 트래픽(80, 443) 허용
- **기본 거부**: 명시적으로 허용되지 않은 모든 트래픽 차단
- **다층 보안**: ACG와 Network ACL 이중 보안 체계

#### Linux iptables 실습 경험:
- **KISA 보안 교육**: 기본 정책을 DROP으로 설정하여 화이트리스트 방식 적용
- **기본 설정**: 로컬 루프백 허용, 기존 연결 유지, 필요한 포트만 선별적 허용
- **상태 추적**: ESTABLISHED, RELATED 연결 상태 기반 트래픽 허용

#### 웹 스크래핑 프로젝트 보안 적용:
- **내부 네트워크 제한**: API 서버는 내부 네트워크에서만 접근 허용
- **데이터베이스 보안**: 특정 서버에서만 DB 포트 접근 가능
- **모니터링 접근**: 관리자 네트워크에서만 모니터링 포트 접근
- **아웃바운드 제어**: 스크래핑용 HTTP/HTTPS 트래픽만 허용
