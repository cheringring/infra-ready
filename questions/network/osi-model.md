## OSI 7계층 모델 상세 설명

OSI(Open Systems Interconnection) 모델은 국제표준화기구(ISO)에서 개발한 네트워크 프로토콜 디자인과 통신을 계층으로 나누어 설명한 것입니다.

### 각 계층별 설명

#### 1. 물리 계층 (Physical Layer)
- 실제 장치들을 연결하기 위한 전기적, 물리적 세부사항 정의
- 비트 스트림을 전기 신호로 변환
- 예: 케이블, 리피터, 허브

#### 2. 데이터링크 계층 (Data Link Layer)
- 물리적 연결을 통해 신뢰성 있는 데이터 전송
- MAC 주소를 사용한 통신
- 프레임 단위로 데이터 전송
- 예: 이더넷, 스위치, 브리지

#### 3. 네트워크 계층 (Network Layer)
- 데이터를 목적지까지 가장 안전하고 빠르게 전달 (라우팅)
- IP 주소를 사용한 논리적 주소 지정
- 패킷 단위로 데이터 전송
- 예: IP, 라우터, ICMP

#### 4. 전송 계층 (Transport Layer)
- 종단 간 신뢰성 있는 데이터 전송
- 포트 번호를 사용하여 프로세스 식별
- 흐름 제어, 오류 제어, 혼잡 제어
- 예: TCP, UDP

#### 5. 세션 계층 (Session Layer)
- 통신 세션을 구성하고 관리
- 데이터 교환의 동기화
- 예: NetBIOS, RPC

#### 6. 표현 계층 (Presentation Layer)
- 데이터의 형식 변환, 암호화, 압축
- 응용 계층과 독립적인 데이터 표현
- 예: JPEG, MPEG, SSL/TLS

#### 7. 응용 계층 (Application Layer)
- 사용자와 가장 가까운 계층
- 네트워크 서비스를 응용 프로그램에 제공
- 예: HTTP, FTP, SMTP, DNS

### 실무에서의 활용

실제 네트워크 문제 해결 시 OSI 모델을 기준으로 계층별로 문제를 진단합니다:
- 물리적 연결 문제 → 1-2계층
- 라우팅 문제 → 3계층
- 포트/방화벽 문제 → 4계층
- 애플리케이션 문제 → 7계층
