## TCP vs UDP 상세 비교

### TCP (Transmission Control Protocol)

#### 특징
- **연결 지향적**: 3-way handshake로 연결 수립
- **신뢰성**: 데이터 전송 보장, 순서 보장
- **흐름 제어**: 수신자의 처리 속도에 맞춰 전송
- **혼잡 제어**: 네트워크 상황에 따라 전송 속도 조절
- **오류 검출 및 재전송**: 손실된 패킷 재전송

#### 3-Way Handshake
1. SYN: 클라이언트 → 서버 (연결 요청)
2. SYN-ACK: 서버 → 클라이언트 (연결 수락)
3. ACK: 클라이언트 → 서버 (연결 확립)

#### 사용 사례
- HTTP/HTTPS (웹)
- FTP (파일 전송)
- SMTP (이메일)
- SSH (원격 접속)

### UDP (User Datagram Protocol)

#### 특징
- **비연결형**: 연결 수립 과정 없음
- **빠른 전송**: 오버헤드가 적음
- **신뢰성 없음**: 데이터 전송 보장 안 함
- **순서 보장 안 함**: 패킷이 순서대로 도착하지 않을 수 있음
- **브로드캐스트 지원**: 일대다 통신 가능

#### 사용 사례
- DNS (도메인 조회)
- DHCP (IP 할당)
- 실시간 스트리밍 (영상, 음성)
- 온라인 게임
- VoIP

### 비교표

| 특성 | TCP | UDP |
|------|-----|-----|
| 연결 | 연결 지향 | 비연결형 |
| 신뢰성 | 높음 | 낮음 |
| 속도 | 느림 | 빠름 |
| 순서 보장 | O | X |
| 오류 검출 | O | 제한적 |
| 헤더 크기 | 20바이트 | 8바이트 |

### 선택 기준

**TCP를 선택하는 경우:**
- 데이터 무결성이 중요한 경우
- 순서가 중요한 경우
- 파일 전송, 웹 페이지 로딩 등

**UDP를 선택하는 경우:**
- 속도가 중요한 경우
- 약간의 데이터 손실이 허용되는 경우
- 실시간 통신, 스트리밍 등
