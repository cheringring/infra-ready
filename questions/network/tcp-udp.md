shortAnswer: "TCP는 연결 지향적이고 신뢰성 있는 데이터 전송을 보장하며, UDP는 비연결형으로 빠르지만 신뢰성을 보장하지 않습니다. TCP는 파일 전송, 웹 브라우징 등에 사용되고, UDP는 실시간 스트리밍, 온라인 게임 등에 사용됩니다."

## 상세 답변

### TCP (Transmission Control Protocol)

#### 특징
- **연결 지향적**: 3-way handshake로 연결 수립
- **신뢰성 보장**: 데이터 순서 보장, 재전송 메커니즘
- **흐름 제어**: 수신자의 처리 속도에 맞춰 전송
- **혼잡 제어**: 네트워크 상황에 따라 전송 속도 조절
- **전이중 통신**: 양방향 동시 통신 가능

#### 동작 방식
```
1. 연결 수립 (3-way handshake)
   Client → Server: SYN
   Server → Client: SYN + ACK
   Client → Server: ACK

2. 데이터 전송
   - 순서 번호(Sequence Number)로 순서 보장
   - 확인 응답(ACK)으로 수신 확인
   - 타임아웃 시 재전송

3. 연결 종료 (4-way handshake)
   Client → Server: FIN
   Server → Client: ACK
   Server → Client: FIN
   Client → Server: ACK
```

#### 사용 사례
- **웹 브라우징** (HTTP/HTTPS)
- **이메일** (SMTP, POP3, IMAP)
- **파일 전송** (FTP)
- **원격 접속** (SSH, Telnet)
- **데이터베이스 연결**

### UDP (User Datagram Protocol)

#### 특징
- **비연결형**: 연결 수립 과정 없음
- **신뢰성 미보장**: 데이터 손실 가능, 순서 보장 안 됨
- **빠른 전송**: 오버헤드가 적어 속도가 빠름
- **단순한 구조**: 헤더가 8바이트로 간단
- **브로드캐스트/멀티캐스트 지원**

#### 동작 방식
```
1. 연결 수립 없이 바로 전송
2. 데이터그램 단위로 전송
3. 수신 확인 없음
4. 손실되면 재전송 안 함
```

#### 사용 사례
- **실시간 스트리밍** (YouTube, Netflix)
- **온라인 게임**
- **VoIP** (음성/영상 통화)
- **DNS 조회**
- **DHCP**
- **IoT 센서 데이터**

### 비교표

| 특성 | TCP | UDP |
|------|-----|-----|
| 연결 방식 | 연결 지향 | 비연결형 |
| 신뢰성 | 보장 | 미보장 |
| 순서 보장 | O | X |
| 속도 | 느림 | 빠름 |
| 헤더 크기 | 20바이트 | 8바이트 |
| 재전송 | O | X |
| 흐름 제어 | O | X |
| 혼잡 제어 | O | X |
| 브로드캐스트 | X | O |

### 선택 기준

#### TCP를 선택해야 할 때
- 데이터 무결성이 중요한 경우
- 순서가 중요한 경우
- 신뢰성이 최우선인 경우
- 예: 금융 거래, 파일 다운로드, 이메일

#### UDP를 선택해야 할 때
- 실시간성이 중요한 경우
- 약간의 데이터 손실이 허용되는 경우
- 빠른 응답이 필요한 경우
- 예: 라이브 방송, 게임, DNS

### 실무 예시

**Netflix 스트리밍**
- 초기 연결: TCP (메타데이터, 인증)
- 영상 스트리밍: UDP (QUIC 프로토콜)
- 이유: 몇 프레임 손실보다 끊김 없는 재생이 중요

**온라인 게임**
- 게임 플레이: UDP (캐릭터 위치, 액션)
- 채팅, 아이템 거래: TCP (정확성 필요)

### 면접 팁

신입 면접에서는 단순히 차이점만 말하는 것보다, **실제 사용 사례와 선택 이유**를 함께 설명하면 좋은 인상을 줍니다.

"예를 들어 제가 사용하는 Discord는 음성 통화에 UDP를 사용하지만, 텍스트 메시지는 TCP를 사용합니다. 음성은 실시간성이 중요하고 약간의 노이즈는 괜찮지만, 메시지는 정확하게 전달되어야 하기 때문입니다."
